## 2. Showing JWT (Authentication)
**Goal:** Prove that your authentication is secure and stateless using JSON Web Tokens.

### Step 1: The Login Flow (Network Tab)
1.  Open your Frontend (`http://localhost:5173/login`).
2.  Open **Developer Tools** (Press `F12` or Right Click -> Inspect).
3.  Go to the **Network** tab.
4.  Enter valid credentials (e.g., `user@example.com` / `password`) and click **Login**.
5.  Look for the request named `access-token` in the list.
6.  Click it and go to the **Response** tab.
7.  **Higlight:** You will see a JSON response like:
    ```json
    {
      "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "token_type": "bearer"
    }
    ```
    *   *Explain:* "This is the JWT (Access Token) returned by the server upon successful login. We don't use sessions; we use this token."

### Step 2: Token Storage
1.  Still in Developer Tools, go to the **Application** tab.
2.  On the left sidebar, expand **Local Storage** -> `http://localhost:5173`.
3.  **Point out:** The `token` key with the long string value.
    *   *Explain:* "We store this token securely in the browser to maintain the user's session state."

### Step 3: Protected Routes (Code & Action)
1.  Go to `backend/app/api/deps.py`.
2.  Show the `get_current_user` function:
    ```python
    def get_current_user(token: str = Depends(reusable_oauth2)) -> User:
        payload = jwt.decode(token, settings.SECRET_KEY, algorithms=[security.ALGORITHM])
    ```
    *   *Explain:* "This dependency inspects every request. If the token is missing or invalid, it rejects the request before it even reaches the route logic."
